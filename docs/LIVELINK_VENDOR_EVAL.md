# LiveLink Vendor Evaluation

> Evaluating the real-time avatar and voice stack for nuncio's conversational SDR.

## Executive Summary

Nuncio's strategic pivot is from "AI-powered personalized video" to a **conversational SDR**: an AI avatar of the sender that prospects can talk to, negotiate with, and book meetings with. Recorded HeyGen videos remain the scalable wedge; the **live avatar session** is the conversion surface.

This doc evaluates the two layers required for a LiveLink prototype:

1. **Avatar / face layer** — real-time video of the sender's likeness.
2. **Voice / audio layer** — low-latency TTS and STT for natural turn-taking.

## Use-Case Requirements

| Requirement | Target | Why |
|-------------|--------|-----|
| End-to-end turn latency | < 800 ms | Above ~800 ms the conversation feels broken and unnatural. |
| Custom avatar | Sender's face + voice | Trust and identity; user feedback specifically asks for "their own face." |
| Sender playbook enforcement | Hard constraints + wiggle room | The agent must stay within guardrails (offer, constraints, flexibility). |
| Web-first delivery | Browser / mobile | Prospects click a link and talk; no app install. |
| Idiomatic integration | Next.js / TypeScript SDK | Fits the existing stack. |
| Pricing predictability | Per-minute or per-character, not surprise enterprise sales | Hackathon / indie builder budget. |

## Avatar Layer

### Anam.ai

**Product overview**
- Purpose-built for real-time, interactive, photorealistic avatars.
- Generates lip-synced video from an audio stream produced by LLM/TTS.
- Uses the **CARA-4** diffusion model; one-shot avatar creation from a single photo.
- First-class plugins for **LiveKit**, **Pipecat**, **Agora**, and **VideoSDK**.

**Latency**
- Server-side avatar generation: ~**150 ms** once a session is initialized.
- End-to-end turn-taking target: sub-**900 ms**.

**Pricing**
- Usage-based per-minute streaming; plans include a free tier and scale by concurrent sessions / custom-avatar slots.
- Primary scaling bottleneck is **concurrent sessions**, not rendered minutes.

**Integration**
- JavaScript / Python SDKs.
- Works best when the audio stream is already generated by a separate TTS; Anam focuses on the face, not the brain.

**Pros**
- Fastest path to a low-latency custom face.
- Single-photo avatar creation matches the "upload your own face" requirement.
- Developer-first; plug-and-play with LiveKit/Pipecat.

**Risks**
- Still an emerging vendor; long-term pricing and runway are less certain than HeyGen.
- Single-photo avatars may look less "alive" than video-trained avatars.
- Concurrency limits can become a hard scaling ceiling.

### HeyGen LiveAvatar

**Product overview**
- Real-time, two-way interactive AI avatar platform hosted at `liveavatar.com`.
- WebRTC streaming.
- Two modes:
  - **Full Mode**: HeyGen handles ASR → LLM → TTS.
  - **Lite Mode**: bring your own LLM/voice stack; HeyGen is just the video engine.
- Custom avatars require video footage (or photo-based avatars depending on plan).

**Latency**
- Markets itself as "one of the fastest in the market" but does not publish firm ms benchmarks.
- Full Mode 1080p is reportedly slower than Anam; Lite Mode is faster because you control the rest of the pipeline.

**Pricing**
- Separate credit system from main HeyGen subscription.
- Starter (~$19/mo) up to Business (~$475/mo).
- Roughly **$0.10–$0.20 / minute** of streaming, depending on mode and plan.

**Integration**
- Web SDK or iframe URL embed.
- Pre-built "Agent Skills" for LLM/orchestration.
- Works with LiveKit / Pipecat.

**Pros**
- Existing HeyGen ecosystem; we already use HeyGen for recorded video.
- Broadcast-quality realism when video footage is supplied.
- SOC 2 compliance available.

**Risks**
- Video footage requirement is more friction than a single photo.
- Full Mode is a black box and likely slower; Lite Mode requires building the rest of the pipeline anyway.
- Highest per-minute cost of the avatar options.

### Avatar Recommendation

**Start with Anam.ai** for the LiveLink prototype.

- It is the only option optimized first for real-time conversation, not for polished pre-rendered video.
- Single-photo avatar creation matches the user request to "put in their own face."
- The latency story is explicit and competitive.

Keep **HeyGen LiveAvatar Lite** as a fallback if the existing HeyGen relationship, compliance requirements, or visual fidelity become deciding factors.

## Voice Layer

### ElevenLabs (current)

**What we already use**
- TTS for soundscapes, cinematic entrances, and script previews.
- Speech Engine for the existing voice brief overlay.
- Voice cloning is already supported.

**Latency**
- Flash model: ~**75 ms**.
- Full model: **300 ms+**.

**Pricing**
- Pay-as-you-go; already budgeted in nuncio.

**Pros**
- Already integrated; no new vendor relationship.
- Excellent voice cloning and emotional expressiveness.

**Risks**
- Not the absolute fastest for real-time conversation.
- Cost can scale quickly with high-throughput live sessions.

### Cartesia

**Product overview**
- **Sonic**: streaming TTS.
- **Ink**: streaming STT with native turn detection and semantic endpointing.
- Built on **State Space Models (SSMs)** rather than transformers, optimized for real-time agents.

**Latency**
- Sonic Turbo: ~**40 ms** model latency.
- Ink provides native VAD and `turn.eager_end` to start LLM inference before the user finishes speaking.

**Pricing**
- Free: 20K credits/mo.
- Pro: $5/mo (100K credits).
- Startup: $49/mo (1.25M credits).
- Scale: $299/mo (8M credits).
- Phone agents: $0.06/min + $0.014/min for numbers.

**Pros**
- Purpose-built for low-latency voice agents.
- Unified STT + TTS reduces vendor stitching.
- Cheaper than ElevenLabs for high-throughput real-time use.

**Risks**
- Newer vendor; less community and fewer battle-tested integrations than ElevenLabs.
- Voice-cloning quality may not yet match ElevenLabs for expressive sales conversation.
- Another API key, billing relationship, and integration surface.

### Voice Recommendation

**Evaluate Cartesia Sonic/Ink in parallel, but default to ElevenLabs for the first LiveLink build.**

- ElevenLabs is already wired into nuncio and has the best voice-cloning quality today.
- If end-to-end latency is > 800 ms in practice, swap the TTS path to Cartesia Sonic as a drop-in replacement.
- Cartesia Ink is a nice-to-have for turn detection but not required if the existing WebRTC/orchestrator layer handles VAD well.

## Proposed Stack

| Layer | Primary | Fallback |
|-------|---------|----------|
| Avatar | Anam.ai | HeyGen LiveAvatar Lite |
| TTS | ElevenLabs | Cartesia Sonic |
| STT | ElevenLabs / WebRTC VAD | Cartesia Ink |
| Orchestration | Pipecat or LiveKit | Custom WebRTC + WebSocket |
| LLM | Existing LLM fallback chain | — |
| Face data | Single photo upload | HeyGen photo avatar |

## End-to-End Latency Budget

| Stage | Budget | Notes |
|-------|--------|-------|
| STT | 100–200 ms | ElevenLabs or Cartesia Ink. |
| LLM inference | 300–500 ms | Use fast model; stream first sentence. |
| TTS first byte | 50–300 ms | ElevenLabs Flash or Cartesia Sonic. |
| Avatar render | ~150 ms | Anam.ai server-side. |
| Network overhead | 50–100 ms | WebRTC / SFU proximity. |
| **Total** | **600–900 ms** | Must stay under ~800 ms to feel natural. |

The biggest wins will come from:
- Streaming the LLM response sentence-by-sentence so TTS/avatar can start before the full response is ready.
- Using `turn.eager_end` (Cartesia) or an aggressive VAD to begin LLM inference during the user's final words.
- Keeping the avatar session warm (pre-initialized) so the 150 ms Anam render cost is only paid once.

## Integration Risks

1. **Avatar-session cost control.** Live avatar minutes are consumed while the prospect is on the page, even if silent. Need per-session credit guards and idle-timeout logic.
2. **Guardrails in real time.** The sender playbook must be enforced inside the live prompt; the avatar cannot promise pricing or terms outside the playbook.
3. **Fallback when live fails.** If WebRTC or avatar service errors, fall back to a static share page or async video.
4. **Privacy / consent.** Recording or processing a prospect's voice requires clear consent and a way to delete data.
5. **Mobile browser support.** WebRTC + real-time avatar on mobile Safari is historically flaky; test early.
6. **Custom-avatar IP.** The sender owns their likeness; terms should not grant broad rights to the vendor.

## Recommendation

**For the LiveLink MVP:**

1. **Build with Anam.ai + ElevenLabs.** This gives the fastest path to a working real-time conversation using the sender's face and voice, without needing new voice-vendor integration.
2. **Use the existing nuncio pipeline** to produce the profile, script, and Sender Playbook; feed those into the live session's system prompt.
3. **Prototype with one test sender** and 5–10 prospects to measure end-to-end latency and conversion to booked meetings.
4. **If latency > 800 ms**, swap TTS to Cartesia Sonic; if cost or realism is an issue, evaluate HeyGen LiveAvatar Lite.

The strategic bet is that a prospect will book a meeting faster when they can talk back than when they watch a recording. This stack lets us test that bet in weeks, not months.
